1. 文件目的與使用情境

本平台供連鎖品牌於加盟展展示展店情形使用。核心情境:展會業務向潛在加盟主展示集團各品牌現有分店與保障範圍,即場查詢潛在加盟主屬意的店址是否落入既有保障範圍,並在其付訂金後將該點位登錄為潛在點位,形成準保障範圍,避免重複招商。

1.1 使用角色

角色說明V1.0 主要操作
展會業務加盟展現場操作者,同一展位僅一個新增窗口圖層瀏覽、地址查詢、潛在點位增刪查改
總部管理者維護基礎資料品牌/分店資料維護、保障範圍參數設定(V1.0 可先以資料匯入方式處理,維護介面於後續版本補齊)
V1.0 不揭露敏感資訊,故不區分展示模式/作業模式;展示與作業共用同一介面,權限與模式分離規劃於 V2.0。

2. 版本藍圖

版本功能項目

4. V1.0 功能規格

F-01 地圖框架與圖層開關

功能說明以地圖為主畫面,所有資訊(分店、保障範圍、潛在點位、參考圖層)均以獨立圖層呈現,可個別開關、自由疊加。
操作流程進入系統載入底圖與預設圖層(分店+保障範圍)→ 於圖層面板勾選/取消圖層 → 地圖即時重繪。
業務規則(1) 品牌篩選:可選「全部品牌」或勾選單一/多個品牌,篩選同時作用於分店、保障範圍、潛在點位圖層。(2) 各品牌以固定代表色區別。(3) 圖層開關狀態不需跨登入保存(V1.0)。
例外處理圖層資料載入失敗時,該圖層顯示錯誤標記並可重試,不影響其他圖層。

F-02 品牌與分店管理

功能說明維護集團-品牌-分店三層資料。V1.0 以批次匯入(Excel/CSV)+基本編輯為主。
資料項目分店:所屬品牌、店名、地址、座標(匯入時地理編碼)、所在行政區、店型(街邊店/百貨店/車站店/賣場店/其他)、是否計入保障範圍(布林,特殊店型預設「否」,可逐店調整)、營運狀態(營運中/籌備中/已歇業)。
業務規則(1) 僅「營運中/籌備中」且「計入保障範圍=是」的分店產生保障範圍。(2) 地址變更時重新地理編碼並重算所在行政區。
例外處理匯入時地理編碼失敗的分店列入待處理清單,可於地圖手動定位。

F-03 保障範圍

功能說明依品牌設定之模式繪製各分店保障範圍:行政區模式(分店所在行政區整區為保障範圍)或半徑模式(以分店座標為圓心、X 公里為半徑)。
設定層級(1) 模式與參數(半徑公里數)設定於品牌層級,同品牌各分店原則一致。(2) 個別分店可覆寫模式或參數。(3) 跨品牌互斥:預設不互斥(A 品牌保障範圍不阻擋 B 品牌進駐),提供品牌兩兩互斥之設定彈性。
業務規則(1) 行政區模式以「鄉鎮市區」為單位,採內政部 TGOS 行政區圖資並記錄圖資版本。(2) 半徑模式以測地線距離(geodesic)計算。(3) 「已付訂保留」之潛在點位依其所屬品牌模式產生準保障範圍,與正式保障範圍同樣參與衝突檢核,圖面以虛線區別。
例外處理行政區圖資版本更新造成分店行政區歸屬變動時,列出差異清單供管理者確認。

F-04 潛在店址查詢與衝突檢核

功能說明輸入詳細地址、路段或行政區進行定位,檢核該落點是否位於任何有效保障範圍(含準保障範圍)內,並可檢視周邊店家分佈。
操作流程輸入地址 → 地理編碼定位 → 系統對「勾選中的品牌」逐一檢核 → 顯示各品牌可展店/不可展店結果與衝突來源(哪一家分店/點位的保障範圍)→ 可一鍵將該落點「新增為潛在點位」。
檢核邏輯(1) 行政區模式品牌:落點行政區 = 任一有效分店所在行政區 → 衝突。(2) 半徑模式品牌:落點與任一有效分店距離 ≦ 該店半徑 → 衝突。(3) 準保障範圍(已付訂保留點位)一併檢核。(4) 若設定跨品牌互斥,則互斥品牌之保障範圍亦計入。(5) 輸入路段/行政區(非門牌)時,以代表點定位並於結果標註「概略位置,建議確認詳細地址」。
例外處理地址解析失敗 → 引導使用者直接於地圖點選落點,點選後執行相同檢核流程(V1.0 必備)。

F-05 潛在點位管理(增刪查改)

功能說明展會現場將付訂之預定店址登錄為潛在點位。V1.0 僅需增刪查改,不做有效期管理。
資料項目點位名稱、所屬品牌、地址/座標、所在行政區、狀態(查詢中/已付訂保留)、聯絡備註、建立時間、最後異動時間。
業務規則(1) 狀態為「已付訂保留」即產生準保障範圍(見 F-03)。(2) 刪除採確認後實刪;完整狀態機(已簽約/已開店/已釋出)與歷程保留於 V2.0 實作。(3) 同一展位單一新增窗口,V1.0 不處理多設備併發鎖定(V2.0)。
例外處理新增時若落點已在同品牌保障範圍內,顯示警示但允許儲存(標記「衝突待確認」),由業務判斷。

F-06 參考圖層:租金行情

功能說明以每月租金為級距(Range)渲染面量圖(choropleth/網格),顏色深淺代表行情高低,附圖例。
業務規則(1) 級距與配色可由設定檔調整。(2) 點擊區塊顯示租金區間與資料時間。(3) 資料來源待評估(實價登錄租賃資料為候選),更新頻率依來源而定(建議季更)。

F-07 參考圖層:店家 POI

功能說明串接外部 POI 資料,顯示周邊店家分佈(餐飲、零售、超商、競品等),供展店評估快速瀏覽。
業務規則(1) 依地圖範圍與縮放層級動態載入,設定最低縮放層級避免全圖大量點位。(2) 依類別篩選與配色。(3) 點擊顯示店名、類別、地址。(4) 來源待評估(Google Places 計費/OSM/政府開放資料混用為候選);若採計次計費來源,查詢結果快取。

F-08 參考圖層:大眾運輸點位與步行距離

功能說明顯示捷運站、公車站等大眾運輸點位;對查詢落點或分店,以 Google 路徑規劃(步行)計算至鄰近運輸點之步行距離與時間並繪製路徑。
業務規則(1) 路徑結果落地快取:以「起點座標(網格化)+目的點位+運具」為鍵,已存在即直接取用,不重複呼叫 API。(2) 預設僅計算最近 N 站(N 可設定,預設 3)。(3) 顯示距離(公尺)與步行時間(分鐘)。
例外處理API 呼叫失敗時顯示直線距離並標註「非步行路徑」。

F-09 參考圖層:停車場與步行距離

功能說明同 F-08 邏輯,對象為停車場點位(公有/民營)。
業務規則與 F-08 共用路徑快取服務與快取規則。

F-10 參考圖層:人口與客群統計

功能說明以政府開放資料(如 SEGIS)之村里級統計渲染面量圖:人口密度、年齡結構、所得中位數等指標可切換。
業務規則(1) 一次僅顯示一個統計指標。(2) 點擊顯示該村里指標明細與資料年度。(3) 與租金圖層互斥顯示(同為面量渲染,避免混淆)。

10. V1.1 / V2.0 功能概述

V1.1

等時圈商圈範圍:對分店或查詢落點產生步行 5/10 分鐘實際可達範圍(isochrone),與半徑保障範圍並列參考。候選點評分卡:查詢落點自動彙整租金區間、周邊 POI 統計、最近運輸點步行時間、人口指標為單頁摘要。操作記錄:記錄查詢地址、新增/異動點位之操作者與時間,作為展會後潛在客戶(leads)名單來源。

V2.0

完整點位狀態機(已簽約/已開店/已釋出)與有效期管理、多設備即時同步(WebSocket)、一鍵點位評估報告(PDF/docx)、展示模式與作業模式分離及角色權限、推薦展店區域維護後台(人工圈選+推薦理由)、API 用量儀表板與費用熔斷。
